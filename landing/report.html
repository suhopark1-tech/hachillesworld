지금 위치의 의미

L2.1 × Digital Laws는 멀티스텝 계획을 시작했지만 아직 초기 단계임을 의미합니다. 에이전트가 "만약 이 행동을 하면 어떻게 될까?"라고 3~5단계 앞을 시뮬레이션할 수 있지만, 그 시뮬레이션의 정확도와 깊이가 아직 부족합니다.

목표인 L2.5 → L3로 가려면 먼저 현재 L2의 품질(예측 정확도, 캘리브레이션)을 높여야 합니다. 불안정한 기반 위에 더 높은 층을 쌓으면 전체가 무너집니다.

예측 오차율 🟡

에이전트가 "다음에 무슨 일이 일어날 것"이라고 예측한 것과 실제로 일어난 것의 차이를 측정합니다.

Prediction Error Rate (PER)
측정 기간: 최근 30일 · 1,247개 예측 이벤트 분석
0.18
기준값: < 0.15
0.00 (완벽)← 기준 0.150.50 (심각)

🤔 이게 무슨 뜻인가요?

에이전트가 내리는 모든 결정은 미래 예측을 기반으로 합니다. 예측 오차율 0.18은 100번 중 18번의 의사결정이 현실과 다른 가정 위에 내려진다는 의미입니다.

예를 들어 재고 보충 에이전트라면, 내일 재고가 500개 필요하다고 예측했는데 실제로는 590개가 필요한 상황이 18%의 경우에 발생합니다. 이는 품절이나 과잉 재고로 이어집니다.

기준값 0.15보다 20% 높은 상황으로, 즉각적 위험은 아니지만 이 상태가 지속되면 Simulation Drift(섹션 06)를 악화시킵니다.

개선 방법
  • 4주 이내앙상블 예측 모델 적용 — 5개 독립 모델의 예측 평균으로 오차를 약 30% 감소시킬 수 있습니다.
  • 4주 이내도메인 특화 피처 추가 — 비즈니스 도메인 고유의 패턴(요일·계절성·이벤트 등)을 입력 피처로 추가합니다.
  • 12주온라인 학습 루프 구축 — 예측이 틀렸을 때 자동으로 모델을 업데이트하는 파이프라인을 만듭니다.

캘리브레이션 ECE 🔴

에이전트가 스스로 얼마나 확신하는지와, 실제로 그 확신이 얼마나 맞는지의 일치도를 측정합니다.

Expected Calibration Error (ECE)
낮을수록 좋음 · 과신 유형: Overconfidence
0.23
기준값: < 0.10
0.00 (완벽)← 기준 0.100.50 (위험)

🤔 이게 무슨 뜻인가요?

ECE 0.23은 심각한 과신(Overconfidence) 문제를 나타냅니다. 쉽게 말해, 에이전트가 "나는 80% 확신한다"고 말할 때 실제로는 약 57%만 맞습니다.

이것이 위험한 이유: 에이전트는 자신이 틀릴 수 있다는 사실을 모르고 자신 있게 결정을 내립니다. 불확실한 상황에서 사람에게 검토를 요청해야 할 때 요청하지 않고, 스스로 틀린 결정을 내립니다.

비유하자면, 운전 경험이 1년 된 사람이 20년 베테랑처럼 자신 있게 운전하는 상황입니다. 기준값(0.10)의 2.3배 수준으로, 즉각적인 조치가 필요합니다.

# 캘리브레이션 교정을 위한 Temperature Scaling 적용 예시 from torch import nn class TemperatureScaler(nn.Module): def __init__(self): super().__init__() # Temperature = 1.0 이 기준. 과신 시 1.0 이상으로 학습됨 self.temperature = nn.Parameter(torch.ones(1) * 1.5) def forward(self, logits): # 확신도를 현실에 맞게 조정 return logits / self.temperature
개선 방법
  • 즉시Temperature Scaling 적용 — 기존 모델을 수정하지 않고 출력층에만 추가하는 가장 빠른 방법입니다. 1~2일 내 구현 가능합니다.
  • 2주불확실성이 높을 때 자동으로 HITL(사람 검토) 트리거 추가 — 캘리브레이션이 개선될 때까지 안전망 역할을 합니다.
  • 4주앙상블 예측으로 전환 — 여러 모델의 의견 불일치가 곧 불확실성이 됩니다.

Simulation Drift 🔴

에이전트 내부의 세계 모델이 실제 환경의 변화를 따라가지 못하고 괴리되는 정도를 측정합니다.

Recalibration Rate (재보정 필요 비율)
허용치의 3.1배 수준 · 가장 시급한 문제
31%
기준값: < 10%
0% (안정)← 기준 10%50% (위험)

🤔 이게 무슨 뜻인가요?

에이전트는 스스로 세계를 이해하는 내부 지도(World Model)를 갖고 있습니다. 실제 세계가 변할 때 이 지도도 업데이트돼야 합니다. Simulation Drift 31%는 10번 중 3번 이상의 상황에서 에이전트의 지도가 현실과 다르다는 뜻입니다.

이 수치는 이번 주 실제 로그를 분석한 결과이며, 지난달보다 8%p 증가했습니다. 추세가 악화 중이므로 지금 즉시 조치하지 않으면 한 달 내에 40%를 넘을 수 있습니다.

근본 원인: 재보정 임계값이 너무 낮게 설정돼 있어(현재 0.08) 실제로 필요한 상황에도 재보정이 트리거되지 않습니다. 임계값을 0.12로 완화하는 것만으로도 즉각적인 개선이 가능합니다.

  • 즉시 (1일)재보정 임계값 0.08 → 0.12로 조정 — 과도하게 잦은 재보정이 드리프트 수치를 왜곡하고 있습니다.
  • 즉시 (3일)드리프트 감지 파이프라인 구축 — 현재 수동 확인 중인 드리프트를 자동으로 감지하고 알림을 받습니다.
  • 2주환경 Non-stationarity 분석 — 드리프트가 특히 심한 시간대·상황을 파악해 도메인 특화 재보정 전략을 수립합니다.

불확실성 커버리지 🟡

에이전트가 "모른다"고 표현하는 범위가 실제 오차를 얼마나 포괄하는지 측정합니다.

Uncertainty Coverage Rate
높을수록 좋음 · 목표: 실제 오차의 80% 이상 포함
68%
기준값: > 80%
0% (불량)기준 80% →100% (완벽)

🤔 이게 무슨 뜻인가요?

좋은 에이전트는 자신이 틀릴 수 있는 범위를 정직하게 표현합니다. 68%는 실제로 오차가 발생한 상황의 32%를 에이전트가 미리 "불확실하다"고 표시하지 않았다는 의미입니다.

이것이 ECE 과신 문제(섹션 05)와 직접 연결됩니다. 에이전트가 불확실한 상황을 불확실하다고 인식하지 못하면, 사람 검토 없이 틀린 결정을 내리게 됩니다.

  • 4주앙상블 모델 예측 분산을 불확실성 지표로 활용 — 모델들의 의견이 갈릴수록 불확실성이 높다고 표시합니다.
  • 4주Conformal Prediction 적용 — 통계적으로 보장된 불확실성 범위를 자동으로 계산합니다.

장기 시뮬레이션 안정성 🟡

에이전트가 멀리 앞을 내다볼수록 예측이 얼마나 빠르게 틀려지는지 측정합니다.

N-step Rollout Error Growth Rate
×2.4
10스텝 후 오차 배율 (기준: <×2.0)

🤔 이게 무슨 뜻인가요?

에이전트가 10단계 앞을 시뮬레이션할 때 예측 오차가 1단계 예측의 2.4배로 커집니다. 이상적으로는 2배 이내여야 합니다. 멀리 볼수록 예측이 빠르게 흐릿해진다는 의미로, 장기 계획의 신뢰도가 낮습니다.

이것이 계획 깊이(섹션 09)가 3스텝에 머무는 이유 중 하나입니다. 더 멀리 보려 해도 오차가 너무 커져 의미가 없습니다.

  • 12주계층적 추상화 적용 — 먼 미래는 큰 그림으로, 가까운 미래는 세부적으로 예측하는 구조로 전환합니다.
  • 12주불확실성 기반 계획 조기 종료 — 불확실성이 임계값을 넘으면 계획을 멈추고 재관측 후 재계획합니다.

계획 깊이 (Planning Depth) 🟡

에이전트가 현재 상황에서 몇 단계 앞까지 내다보고 결정을 내리는지 측정합니다.

평균 계획 깊이 (스텝 수)
3
L2 목표: 15+ 스텝
1 스텝 (L1)L2 목표 15+ →50+ 스텝 (L3)

🤔 이게 무슨 뜻인가요?

현재 에이전트는 체스에서 3수 앞만 보는 플레이어와 같습니다. L2 Simulator 수준의 에이전트는 최소 15수 앞을 계산해야 합니다. 목표의 20% 수준으로, 복잡한 태스크에서 근시안적 결정을 내리고 있습니다.

계획 깊이가 낮으면 "지금 당장은 좋아 보이지만 나중에 문제가 되는" 결정을 자주 내립니다. 공급망 예시라면, 단기 비용을 줄이려 주문을 미루다가 다음 주 재고 부족이 발생하는 패턴입니다.

  • 4~8주MCTS (Monte Carlo Tree Search) 계획기 구현 — 가장 효과적인 방법으로, 실제 시뮬레이션을 통해 최적 행동을 탐색합니다. 책의 Ch.12 코드 참고.
  • 4주계획 깊이를 단계적으로 확장 — 3→5→10→15스텝 순서로 안정성을 확인하며 늘립니다.
