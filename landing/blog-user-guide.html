HAchillesWorld를 처음 접하는 분들이 가장 많이 묻는 것은 "어디서부터 시작하면 되나요?"다.

이 글은 그 질문에 단계별로 답한다. 설치부터 첫 진단 리포트 해석, Optimize 로드맵 생성, 실시간 Drift 모니터링 연결까지 — 각 단계의 명령어와 출력 결과를 그대로 담았다.

이 가이드는 사용자 가이드 v2.1.0 기준이다. v2.1의 주요 신규 기능 — HAS 신뢰구간, HASInterpreter, 영구 스토리지, LocalJudge — 을 포함한다.

② 로그 수집 — SDK 연동

HAchillesWorld가 진단할 수 있는 데이터는 두 가지다. 기존 JSON 로그가 있다면 바로 ③으로 건너뛰어도 된다. 없다면 SDK 데코레이터 하나로 자동 수집을 시작한다.

방법 A: @instrument 데코레이터 (가장 간단)

python — sdk_quickstart.py 
from hachillesworld import HAchillesWorldClient, instrument

client = HAchillesWorldClient()

# 기존 에이전트 클래스에 데코레이터 하나만 붙인다
@instrument(client, agent_name="my-agent")
class MyAgent:
    def plan(self, state, goal):
        # 기존 코드 그대로 유지
        return self._select_action(state, goal)

    def execute(self, action):
        return self._run_action(action)

    def observe(self, result):
        return self._update_state(result)

    def reflect(self, observation):
        self._update_world_model(observation)

plan / execute / observe / reflect 이름의 메서드가 자동으로 계측된다. 다른 메서드는 영향받지 않는다.

방법 B: 개별 이벤트 기록

메서드 이름이 다르거나 세부 정보를 추가로 기록하려면:

python 
import time

client.track(
    agent_name="my-agent",
    event_type="plan",
    payload={
        "planning_depth": 12,     # 몇 스텝 앞을 보는가
        "uncertainty":    0.12,   # 불확실성 수준
        "confidence":     0.88,   # 자기 확신도
    }
)

client.track(
    agent_name="my-agent",
    event_type="observe",
    payload={
        "prediction_error": 0.08,          # 예측-현실 괴리
        "error_within_uncertainty": True,  # 불확실성 구간 내인가
        "goal_achieved": True,
    }
)

client.flush()  # 버퍼 플러시
최소 50개 에피소드를 수집해야 통계적으로 의미 있는 95% 신뢰구간이 산출됩니다. 처음에는 예제 데이터로 시작해도 충분합니다.

④ 결과 해석 — HASInterpreter v2.1 신규

v2.1의 가장 큰 변화 중 하나다. HAS 점수가 나왔을 때 "이게 좋은 건지 나쁜 건지, 다음에 뭘 해야 하는지"를 5초 안에 알 수 있다.

python 
from hachillesworld.interpret.has_interpreter import HASInterpreter

interp = HASInterpreter().interpret(report)

print(f"등급:      {interp.grade} ({interp.grade_label})")
print(f"배포 권고: {interp.deployment_status}")
print(f"산업 평균 대비: 상위 {interp.percentile:.0f}%")
print(f"즉각 조치 시 예상 HAS 향상: +{interp.estimated_improvement:.0f}점")

print("\n[즉각 조치 목록]")
for action in interp.next_actions:
    urgency = ["즉시", "이번 주", "이번 달"][action.priority - 1]
    print(f"  [{urgency}] {action.metric}: {action.current_value:.2f} → {action.target_value:.2f}")
    print(f"           {action.action}")
    print(f"           예상 HAS 향상: +{action.estimated_has_gain:.0f}점")

⑥ Operate — 실시간 모니터링

에이전트가 프로덕션에 배포된 후 World Model이 현실과 멀어지는 것을 실시간으로 추적한다.

Drift 모니터링 연결

python — production_loop.py 
from hachillesworld.operate.monitor import DriftMonitor

monitor = DriftMonitor(
    agent_name="production-agent",
    threshold=0.15,           # 이 값 초과 시 드리프트로 판단
    window_size=20,           # 최근 N개 기록으로 안정성 판단
    alert_rate_threshold=0.20  # 20% 이상이면 경보
)

@monitor.on_alert
def handle_alert(alert):
    print(f"⚡ Drift: {alert.drift_value:.4f}")
    print(f"   권장 조치: {alert.recommended_action}")
    # → Slack 알림, 자동 재보정 트리거 등 연결

# 에이전트 운영 루프 내에서 매 스텝 호출
for step in range(n_steps):
    action    = agent.plan(state, goal)
    result    = agent.execute(action)
    predicted = world_model.predict(state, action)
    actual    = env.observe()

    drift = monitor.record(
        predicted={"value": predicted.value, "cost": predicted.cost},
        actual=   {"value": actual.value,    "cost": actual.cost},
    )

    if drift > monitor.threshold:
        world_model.recalibrate(actual)

summary = monitor.summary()
print(f"경보 횟수: {summary['alert_count']}")
print(f"안정 상태: {summary['is_stable']}")

데이터 보호 체크리스트

HAchillesWorld는 기본적으로 로컬에서만 작동한다. 다음 체크리스트를 확인하자.

  • LocalJudge 사용 확인 — 기본값(HACHILLESWORLD_CA_JUDGE=local)이면 로그가 외부로 나가지 않는다.
  • Privacy Filter 적용 — 로그에 개인식별정보가 포함된 경우 PrivacyFilter로 사전 처리한다.
  • 감사 로그 활성화 — 엔터프라이즈 환경에서는 AuditLogger로 누가 어떤 리포트를 조회했는지 기록한다.
  • AnthropicJudge 사용 시 동의 — CA 측정에 외부 API를 쓰는 경우 조직의 데이터 처리 정책을 확인한다.
python — privacy_example.py 
from hachillesworld.privacy import PrivacyFilter

pf = PrivacyFilter(
    anonymize_fields=["user_id", "email"],     # 해시 처리
    redact_patterns=[r"\d{6}-\d{7}"]            # 주민번호 마스킹
)

sanitized_logs = pf.apply(raw_logs)
report = client.scan(logs=sanitized_logs, config=config, agent_name="my-agent")

알아두면 좋은 팁

01

로그 없이 시작해도 괜찮다

python examples/basic_scan.py만으로 전체 흐름을 경험할 수 있다. 먼저 결과를 보고, 그 다음 SDK를 붙이는 순서가 오히려 빠르다.

02

planning_depth 키가 Level 판정의 핵심

plan 이벤트의 payload에 planning_depth가 없으면 기본값 1이 사용되어 L1으로 낮게 판정된다. 이 키 하나만 추가해도 진단 정확도가 크게 올라간다.

03

95% CI로 개선 효과를 객관적으로 증명한다

개선 전 HAS 72.1 [69.8, 74.4] → 개선 후 HAS 83.5 [81.2, 85.8]. CI가 겹치지 않으면 개선 효과가 통계적으로 유의하다. 경영진 보고에 바로 활용할 수 있다.

04

WMQ가 낮으면 다른 지표를 먼저 보지 말 것

WMQ는 HAS의 45%를 차지한다. WMQ가 60 미만이면 ALM·OHM을 먼저 고치는 것은 순서가 잘못된 것이다. HASInterpreter의 즉각 조치 목록은 항상 우선순위가 올바르게 정렬되어 있다.

05

DriftMonitor threshold는 2주 후 조정한다

처음에는 기본값(0.15)으로 시작한다. 2주간 운영 후 np.percentile(monitor.get_all_drifts(), 95)로 실제 95백분위를 새 임계값으로 설정하면 과잉 경보를 줄일 수 있다.
