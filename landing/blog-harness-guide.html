에이전트가 같은 실수를 두 번 반복하는 것은 용납할 수 없다.

그런데 Harness 없는 에이전트는 정확히 그 일을 한다. 어제 예산 초과 발주를 냈던 에이전트가 오늘도 같은 상황에서 같은 결정을 내린다. 기억이 없기 때문이다.

Harness는 에이전트에게 기억을 준다. 한 번 겪은 실수를 규칙으로 만들어서, 같은 상황이 오면 자동으로 차단한다.

Harness란 무엇인가

Harness는 에이전트 행동을 제약하는 규칙 집합이다. 에이전트가 특정 조건을 만족했을 때 특정 행동을 하거나 하지 못하도록 강제한다.

비유하자면 신입 직원의 결재 라인이다. 신입이 아무리 좋은 판단을 해도, 100만 원 이상 지출은 팀장 결재가 필요하다는 규칙이 있으면 혼자 처리할 수 없다. Harness가 그 역할이다.

● Harness 규칙 구조 
규칙 구성 요소:
  조건 (Condition)  : 언제 이 규칙이 적용되는가
  행동 (Action)     : 무엇을 하거나 막는가
  심각도 (Severity) : warn / block / escalate

예시 규칙:
  조건: action.type == "ORDER" AND amount > budget.remaining
  행동: block
  메시지: "예산 잔액 초과 — 발주 차단"

규칙은 코드가 아니라 선언형 정책으로 작성된다. 에이전트 코드를 수정하지 않아도 Harness 규칙만 추가하면 즉시 적용된다.

설치와 첫 규칙 등록

HAchillesWorld와 함께 Harness를 초기화하는 방법이다.

harness_setup.py 
from hachillesworld import HAchillesWorldClient, HarnessRule, instrument

client = HAchillesWorldClient()
harness = client.harness(agent_name="supply-agent")

# 첫 번째 규칙: 예산 초과 발주 차단
harness.add_rule(HarnessRule(
    name="budget-guard",
    condition="action.type == 'ORDER' and action.amount > state.budget_remaining",
    action="block",
    message="예산 잔액 초과 — 발주 차단됨",
    severity="block"
))

@instrument(client, agent_name="supply-agent")
class SupplyAgent:
    def plan(self, state, goal): ...
    def execute(self, action): ...

이제 에이전트가 예산을 초과하는 발주를 시도하면 자동으로 차단된다. 에이전트 코드는 한 줄도 바꾸지 않았다.

Harness 규칙 유형 4가지

1. Guard 규칙 — 행동 차단

가장 기본적인 유형이다. 특정 조건에서 행동을 완전히 막는다. 예산 초과, 중복 발주, 권한 없는 작업 등에 사용한다.

2. Fallback 규칙 — 대안 행동 제공

행동을 차단하는 대신 안전한 대안을 실행한다. 외부 API가 타임아웃되면 캐시된 값으로 폴백하는 식이다.

3. Escalate 규칙 — 인간 승인 요청

에이전트가 판단하기 어려운 상황에서 인간에게 넘긴다. 고위험 행동, 새로운 상황, 불확실성이 높은 결정에 사용한다.

4. Log 규칙 — 감사 추적

행동을 차단하지는 않지만 자세한 로그를 남긴다. 패턴 분석과 향후 규칙 개선에 사용한다.

harness_rules.py 
# Fallback: API 타임아웃 → 캐시 폴백
harness.add_rule(HarnessRule(
    name="api-timeout-fallback",
    condition="error.type == 'TimeoutError'",
    action="fallback",
    fallback_value="cache.last_known_price"
))

# Escalate: 고액 발주 → 인간 승인
harness.add_rule(HarnessRule(
    name="high-value-escalate",
    condition="action.amount > 5_000_000",
    action="escalate",
    notify="manager@company.com"
))

Meta-Harness: 규칙을 스스로 만드는 에이전트

규칙을 수동으로 등록하는 것은 시작점이다. HAchillesWorld의 Meta-Harness는 에이전트의 오류 패턴을 분석해서 새로운 규칙을 자동으로 제안한다.

meta_harness.py 
# Meta-Harness 활성화
meta = client.meta_harness(agent_name="supply-agent")

# 최근 30일 로그에서 패턴 자동 감지
suggestions = meta.analyze(
    logs="./agent_logs.json",
    lookback_days=30,
    min_frequency=3   # 3회 이상 반복된 오류만
)

for s in suggestions:
    print(f"제안: {s.name}")
    print(f"  조건: {s.condition}")
    print(f"  근거: {s.frequency}회 반복, 손실 {s.estimated_loss:,}원")
    # 승인 시 즉시 적용
    if input("적용? (y/n): ") == "y":
        harness.add_rule(s.to_rule())
● Meta-Harness 자동 제안 예시 
자동 감지된 패턴 8건:

  MA-AUTO-001: 재고 임계치 불일치 발주
    조건: item.stock > threshold_b AND pending_order EXISTS
    빈도: 23회/월, 추정 손실 ₩340만/월
    → 승인? y ✅

  MA-AUTO-002: 공휴일 배송 불가 발주
    조건: delivery.date IS holiday AND region == "domestic"
    빈도: 7회/월, 추정 손실 ₩80만/월
    → 승인? y ✅

  ... (6건 더)

승인된 규칙은 즉시 Harness에 등록되어 동일 패턴 발생 시 자동 차단된다. 에이전트가 점점 영리해지는 것이 아니라, Harness가 점점 촘촘해지는 것이다.
