ECE Calibration 개선 계획

현재 ECE 0.23은 에이전트가 과신하고 있음을 의미합니다. Temperature Scaling으로 2주 안에 0.08 이하로 개선합니다.

왜 ECE가 중요한가

ECE(Expected Calibration Error)가 0.23이면 에이전트가 "80% 확신"한다고 말할 때 실제로는 57%만 맞습니다. 이 과신이 잘못된 결정을 만들고, 특히 공급망처럼 비용이 큰 환경에서 손실을 키웁니다. ECE 0.08 달성 시 "80% 확신 = 실제 79%" 수준이 됩니다.

Step 1 — Temperature Scaling (Week 1, 2일)

calibration_fix.pyPython
from hachillesworld.optimize.calibration import TemperatureScaling import numpy as np # 1. 검증 세트에서 최적 Temperature T 탐색 calibrator = TemperatureScaling() T_optimal = calibrator.fit( logits=validation_logits, # 모델 출력 로짓 labels=validation_labels, method="nll_minimization" # NLL 최소화로 T 탐색 ) print(f"최적 Temperature: {T_optimal:.3f}") # 예: 1.82 # 2. 예측 시 T로 나눠 보정 def calibrated_predict(logits, T=T_optimal): scaled = logits / T probs = softmax(scaled) return probs # 3. 보정 전후 ECE 비교 ece_before = calibrator.compute_ece(raw_probs, labels) ece_after = calibrator.compute_ece(calibrated_predict(logits), labels) print(f"ECE: {ece_before:.3f} → {ece_after:.3f}") # 예상 출력: ECE: 0.230 → 0.081

Self-Correction Rate 개선

현재 자기 수정율 0% — 오류 발생 시 그대로 진행합니다. 3단계 오류 감지·수정 루프로 10%+ 달성합니다.

Self-Correction이 없으면

에이전트가 잘못된 API 응답을 받거나 예상치 못한 상태를 만났을 때, 오류를 인식하지 못하고 그대로 다음 단계로 진행합니다. DEOR(Decide-Execute-Observe-Reflect) 루프에서 Reflect 단계가 사실상 비어 있는 상태입니다.

3단계 자기수정 루프 구현

self_correction_loop.pyPython
from hachillesworld.harness import SelfCorrectionPolicy from hachillesworld.operate.monitor import DriftMonitor policy = SelfCorrectionPolicy( max_retries=3, confidence_threshold=0.65, # 이 이하면 재시도 error_patterns=[ "unexpected_state", "api_timeout", "constraint_violation", ] ) def reflect_and_correct(episode, observation, expected): # Step 1: 오류 감지 error = policy.detect_error(observation, expected) if not error: return observation # 정상 — 그대로 진행 # Step 2: 오류 분류 및 수정 전략 선택 strategy = policy.select_correction(error) print(f"[Self-Correction] {error.type} → {strategy.name}") # Step 3: 수정 실행 (최대 3회) for attempt in range(policy.max_retries): corrected = strategy.apply(episode, error) if policy.validate(corrected): episode.record_self_correction() # 수정율 집계 return corrected # 3회 실패 → HITL 에스컬레이션 return policy.escalate(episode)

MCTS 계획기 통합

Monte Carlo Tree Search로 Planning Depth를 3스텝에서 15스텝 이상으로 확장합니다. 생성된 mcts_planner.py를 그대로 사용하세요.

MCTS가 필요한 이유

현재 에이전트는 탐욕적(greedy) 방식으로 매 스텝 최선을 선택하지만, 3스텝 이후를 보지 못합니다. 공급망처럼 지연 효과가 큰 환경에서 단기 최적이 장기 최악이 되는 경우가 잦습니다. MCTS는 50회 시뮬레이션으로 5스텝 이후 결과를 미리 평가하고 최선의 행동을 선택합니다.

  • Week 9: World Model 래퍼 작성 — MCTS가 시뮬레이션에 사용할 환경 모델 인터페이스
  • Week 10: UCB1 탐색 정책 튜닝 — exploration_c 값 최적화 (A/B 테스트)
  • Week 11: 롤아웃 정책 학습 — random → lightweight neural 업그레이드
  • Week 12: 통합 테스트 및 Planning Depth 15스텝 달성 검증

Counterfactual 추론 레이어

"다른 결정을 했다면 어떤 결과가 나왔을까?" — 반사실 시뮬레이션으로 현재 결정의 질을 평가하고 World Model을 강화합니다.

counterfactual_layer.pyPython
from hachillesworld.optimize.counterfactual import CounterfactualSimulator simulator = CounterfactualSimulator( world_model=agent.world_model, n_alternatives=5, # 대안 행동 5개를 동시 시뮬레이션 horizon=10, # 10스텝 앞까지 시뮬레이션 ) def decide_with_counterfactual(state, proposed_action): # 제안된 행동과 대안들을 동시 시뮬레이션 cf_results = simulator.simulate(state, proposed_action) # 후회 최소화(Regret Minimization)로 최선 선택 best_action = simulator.minimize_regret(cf_results) # 반사실 데이터로 World Model 업데이트 (온라인 학습 준비) simulator.update_world_model(cf_results) return best_action

Phase 3 온라인 학습과의 연결

Counterfactual 데이터는 Phase 3의 온라인 학습 루프에 직접 투입됩니다. "실제로 일어나지 않은 일"을 시뮬레이션한 데이터가 World Model을 더 빠르게, 더 적은 실제 실패로 개선합니다.

Meta-Harness 자기수정

실패 에피소드를 자동 분석해 새로운 하네스 규칙을 생성합니다. 인간이 규칙을 수동 작성하는 한계를 넘어섭니다.

meta_harness.pyPython
from hachillesworld.operate import MetaHarness meta = MetaHarness( agent_name="supply-chain-agent", base_harness=SupplyChainHarness(), # Phase 1에서 생성한 하네스 learning_rate=0.01, min_episodes_to_learn=10, # 10번 실패 후 규칙 생성 ) # Meta-Harness는 백그라운드에서 실패를 관찰 @meta.observe def on_episode_end(episode): if episode.failed: pattern = meta.extract_failure_pattern(episode) if meta.is_new_pattern(pattern): new_rule = meta.generate_rule(pattern) print(f"[MetaHarness] 새 규칙 생성: {new_rule.name}") meta.propose_rule(new_rule) # 사람에게 검토 요청

자동 생성 규칙의 검토 프로세스

# harness_rules.py — HAchillesWorld Optimize 자동 생성 # 에이전트: supply-chain-agent | Level: L2.3 × Digital Laws # 생성 기준: Scan 결과 §05 ECE, §06 Self-Correction, §07 Coverage from hachillesworld.harness import HarnessSpec, Rule, RuleResult class SupplyChainHarness(HarnessSpec): """L2.3 → L3 전환을 위한 자동 생성 하네스""" # ── 규칙 01: API 멱등성 (기존) ──────────────────── def check_idempotency(self, action, history): if action.id in history.executed_ids: return Rule.REJECT("중복 API 호출", action="skip") return Rule.PASS() # ── 규칙 07: ECE Calibration 게이트 (신규) ──────── def check_calibration_ece(self, pred_probs, actual): ece = self._compute_ece(pred_probs, actual) if ece > 0.10: # Scan §05: 현재 0.23, 목표 0.08 return Rule.WARN( reason=f"ECE {ece:.3f} > 0.10 — 재보정 권장", action="flag_for_recalibration" ) if ece > 0.20: return Rule.REJECT( reason=f"ECE {ece:.3f} > 0.20 — 즉시 재보정 필요", action="pause_and_recalibrate" ) return Rule.PASS() # ── 규칙 08: Drift Rate 게이트 (신규) ───────────── def check_drift_rate(self, drift_value): if drift_value > 0.25: return Rule.REJECT( reason=f"Drift {drift_value:.3f} 위험 수준", action="halt_and_alert" ) if drift_value > 0.12: return Rule.WARN(reason=f"Drift {drift_value:.3f} 주의") return Rule.PASS() # ── 규칙 09: 불확실성 임계값 게이트 (신규) ──────── def check_uncertainty(self, confidence): if confidence < 0.20: # 0.35 → 0.20 강화 return Rule.REJECT( reason=f"신뢰도 {confidence:.2f} < 0.20", action="request_human_input" ) return Rule.PASS() # ── 규칙 10: Self-Correction 재시도 (신규) ──────── def check_self_correction(self, episode, error): if error and not episode.attempted_correction: return Rule.RETRY( prompt="이전 결과에서 오류를 감지했습니다. 다시 시도하세요.", max_attempts=3 ) return Rule.PASS() # ── 규칙 04 강화: 예산 한도 (강화) ─────────────── def check_budget(self, cost_estimate, budget_remaining): ratio = cost_estimate / max(budget_remaining, 1) if ratio > 0.90: return Rule.REJECT(reason="예산 90% 초과 예상") if ratio > 0.70: return Rule.WARN(reason=f"예산 {ratio*100:.0f}% 소진 예상") return Rule.PASS()
