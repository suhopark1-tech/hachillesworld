## 1. 소개 및 v2.1 변경 사항

### HAchillesWorld란?

HAchillesWorld는 AI 에이전트의 **World Model 품질을 진단하고 최적화하는 플랫폼**이다.

에이전트가 "작동한다"는 것은 확인하기 쉽다.  
에이전트가 *올바른 이유로* 작동하는지는 확인하기 어렵다.  
HAchillesWorld는 두 번째 질문에 답한다.

```
┌─────────────────────────────────────────────────────────────┐
│  Scan      지금 내 에이전트는 어느 수준인가?                │
│  ─────     Levels × Laws 좌표계로 15분 안에 진단           │
├─────────────────────────────────────────────────────────────┤
│  Optimize  다음 단계로 가려면 무엇을 해야 하는가?           │
│  ────────  자동화된 로드맵 + 하네스 코드 생성               │
├─────────────────────────────────────────────────────────────┤
│  Operate   지금 에이전트가 잘 작동하고 있는가?              │
│  ───────   실시간 드리프트 감지 + Replay 디버깅             │
└─────────────────────────────────────────────────────────────┘
```

### v2.1의 주요 변경 사항

| 범주 | 변경 내용 |
|------|-----------|
| **HAS 신뢰성** | 95% 신뢰구간(CI) 추가 · 가중치 버전 관리 (시계열 비교 가능) |
| **결과 해석** | `HASInterpreter` — 점수 → 등급·배포 권고·즉각 조치 3개 자동 생성 |
| **스토리지** | SQLite 내장 + PostgreSQL 지원 — 재시작 후에도 히스토리 유지 |
| **CA 재현성** | LLM-as-Judge 로컬 옵션 · Judge 모델 버전 고정 · 시드 제어 |
| **데이터 투명성** | Privacy Module · 감사 로그(Audit Log) · 데이터 처리 흐름 문서화 |
| **멀티에이전트** | 복수 에이전트 동시 진단 + 조직 수준 포트폴리오 리포트 |
| **컴플라이언스** | EU AI Act / ISO 42001 / K-AI 안전기준 자체 점검 체크리스트 |
| **측정 정확도** | 측정 불가 지표 `exclude` 정책 (고정값 40 대체 방식 제거) |

```python
from hachillesworld import HAchillesWorldClient, instrument

client = HAchillesWorldClient()

@instrument(client, agent_name="my-agent")
class MyAgent:
    def plan(self, state, goal):
        return self._select_action(state, goal)
    
    def execute(self, action):
        return self._run_action(action)
    
    def observe(self, result):
        return self._update_state(result)
    
    def reflect(self, observation):
        self._update_world_model(observation)
```

## 4. Module 1: Scan — 진단

### 4.1 개요

Scan은 에이전트 로그와 설정을 분석해 15개 지표를 계산하고, Levels × Laws 좌표와 종합 점수를 담은 `DiagnosticReport`를 반환한다.

**v2.1 신규**: 측정 불가 지표는 `exclude` 정책으로 처리된다. 해당 지표는 HAS 계산에서 분모 제외되며, `not_measured_metrics` 필드로 확인 가능하다.

### 4.2 기본 사용법

```python
from hachillesworld import HAchillesWorldClient

client = HAchillesWorldClient()

report = client.scan(
    logs=log_data,          # list[dict] — 에이전트 이벤트 로그
    config=agent_config,    # dict       — 에이전트 설정
    agent_name="my-agent"   # str        — 에이전트 이름
)

# 기본 결과
print(report.summary())
print(f"Level: {report.level_label}")          # 예: "L2.3"
print(f"Score: {report.composite_score:.0f}/100")

# v2.1: 신뢰구간 및 버전
if report.has_confidence_interval:
    lo, hi = report.has_confidence_interval
    print(f"95% CI: [{lo:.1f}, {hi:.1f}]")
print(f"가중치 버전: {report.has_score_version}")

# 측정 불가 지표 확인
if report.not_measured_metrics:
    print(f"측정 불가 지표: {report.not_measured_metrics}")

# Critical 이슈
for issue in report.critical_issues:
    print(f"🔴 {issue.name}: {issue.description}")
```

### 4.3 Levels × Laws 프레임워크

```
         Physical  Digital  Social  Scientific
         Laws      Laws     Laws    Laws
         ─────────────────────────────────────
L1       단일 스텝 예측 / 패턴 매칭
Predictor

L2       멀티스텝 롤아웃 / 반사실 추론
Simulator

L3       자율 루프 / 자기 수정
Evolver
```

**Level 표기법**: `L2.3`은 L2 레벨 내 30% 진행을 의미한다(0~9 스케일).

### 4.4 진단 지표 15개

#### Category A (WMQ): World Model 품질 — 가중치 45%

| 약어 | 지표명 | OK | Warning | Critical |
|------|--------|----|---------|----------|
| PER | Prediction Error Rate | < 0.15 | 0.15~0.30 | > 0.30 |
| ECE | Calibration ECE | < 0.10 | 0.10~0.20 | > 0.20 |
| SDR | Simulation Drift Rate | < 5% | 5~20% | > 20% |
| ODR | OOD Detection Rate | > 80% | 60~80% | < 60% |
| PD  | Planning Depth (스텝) | ≥ 5 | 2~5 | < 2 |

#### Category B (ALM): 에이전시 수준 — 가중치 30%

| 약어 | 지표명 | OK | Warning | Critical |
|------|--------|----|---------|----------|
| SCR | Self-Correction Rate | ≥ 10% | > 0% | 0% |
| CA  | Counterfactual Accuracy | > 0.65 | 0.40~0.65 | < 0.40 |
| GAR | Goal Achievement Rate | > 90% | 75~90% | < 75% |
| AS  | Adaptation Speed (스텝) | ≤ 10 | 10~30 | > 30 |
| HC  | Harness Coverage (규칙 수) | ≥ 20 | 5~20 | < 5 |

#### Category C (OHM): 운영 건전성 — 가중치 25%

| 약어 | 지표명 | OK | Warning | Critical |
|------|--------|----|---------|----------|
| WMUL | World Model Update Latency (초) | < 300 | 300~600 | > 600 |
| IRT  | Incident Recovery Time (분) | < 15 | 15~60 | > 60 |
| HITL | Human-in-the-Loop Rate | < 5% | 5~20% | > 20% |
| HR   | Hallucination Rate | < 5% | 5~15% | > 15% |
| SU   | System Utilization | 60~85% | 85~100% | > 100% |

> **도메인별 임계값**: Physical·Social·Scientific Laws는 임계값이 다르다.  
> `src/hachillesworld/scan/domain_thresholds/` 디렉터리의 YAML 파일을 참조.

### 4.5 DiagnosticReport 전체 구조 (v2.1)

```python
@dataclass
class DiagnosticReport:
    agent_name:           str
    level:                Level            # L1 / L2 / L3
    level_progress:       float            # 0.0~1.0
    laws_domain:          LawsDomain

    world_model_quality:  CategoryScore    # WMQ
    agency_level:         CategoryScore    # ALM
    operational_health:   CategoryScore    # OHM

    recommendations:      list[str]
    metadata:             dict

    # v2.1 신규 필드
    has_confidence_interval:   tuple[float, float] | None  # (lower, upper)
    has_score_version:         str                          # "2.1"
    measurement_metadata:      dict                         # 측정 환경 메타데이터
    not_measured_metrics:      list[str]                    # 측정 불가 지표 목록

    # 자동 계산 프로퍼티
    @property
    def composite_score(self) -> float: ...    # HAS 종합 점수
    @property
    def level_label(self) -> str: ...          # 예: "L2.3"
    @property
    def critical_issues(self) -> list[MetricScore]: ...
```

### 4.6 점수 해석 기준 (v2.1)

| HAS 점수 | 등급 | 상태 | 배포 권고 |
|---------|------|------|-----------|
| 90~100 | A+ | 우수 에이전트 | 전면 배포 가능 |
| 80~89  | A  | 양호 에이전트 | 조건부 배포 |
| 70~79  | B  | 개선 필요 | 감독 하 운용 |
| 60~69  | C  | 주의 필요 | 제한적 운영 |
| 0~59   | D  | 위험 에이전트 | 배포 보류 |

> **HAS 95% 신뢰구간**: 점수 차이가 CI 범위 내에 있을 경우 통계적으로 유의하지 않을 수 있다.  
> 예: HAS 72.4 [70.1, 74.7] vs HAS 73.1 [71.5, 74.6] → 실질적 차이 없음.

## 6. Module 3: Operate — 운영

### 6.1 실시간 Drift 모니터링

```python
from hachillesworld.operate.monitor import DriftMonitor

monitor = DriftMonitor(
    agent_name="production-agent",
    threshold=0.15,
    window_size=20,
    alert_rate_threshold=0.20
)

@monitor.on_alert
def handle_alert(alert):
    print(f"⚡ Drift Alert: {alert.drift_value:.4f}")
    print(f"   권장 조치: {alert.recommended_action}")
    # Slack 알림, 자동 재보정 트리거 등 연결

# DEOR 루프에서 매 스텝
for step in range(n_steps):
    action         = agent.plan(state, goal)
    result         = agent.execute(action)
    predicted      = world_model.predict(state, action)
    actual         = env.observe()

    drift = monitor.record(
        predicted={"value": predicted.value, "cost": predicted.cost},
        actual={"value": actual.value, "cost": actual.cost},
    )

    if drift > monitor.threshold:
        world_model.recalibrate(actual)

    if not monitor.is_stable():
        trigger_human_review()

summary = monitor.summary()
print(f"경보 횟수: {summary['alert_count']}  |  안정: {summary['is_stable']}")
```

### 6.2 Replay 디버깅

실패한 에피소드를 단계별로 재생해 근본 원인을 찾는다.

```python
from hachillesworld.operate.replay import ReplayDebugger

events = [
    {"agent_name": "my-agent", "event_type": "plan",
     "timestamp": 1748745600.0, "payload": {"uncertainty": 0.08, "planning_depth": 3}},
    {"agent_name": "my-agent", "event_type": "execute",
     "timestamp": 1748745601.0, "payload": {"action": "update_inventory"}},
    {"agent_name": "my-agent", "event_type": "observe",
     "timestamp": 1748745602.0, "payload": {"prediction_error": 0.42}},  # 이상
    {"agent_name": "my-agent", "event_type": "reflect",
     "timestamp": 1748745603.0, "payload": {"recalibrated": False}},
]

debugger = ReplayDebugger()
session  = debugger.load(episode_id="ep-2026-001", events=events)
debugger.print_session(session)

print(f"근본 원인: {session.root_cause}")
print(f"실패 스텝: Step {session.failure_step}")
```

출력 예시:
```
  ⚡ 근본 원인: Step 2 (observe): Simulation Drift 0.420 > 0.15 — 재보정 미실행 여부 확인
  
  🔴 Step 2 [observe]: prediction_error = 0.42
  Step 3 [reflect]: recalibrated = False  ← 재보정 미실행
```

### 6.3 Meta-Harness 자동 학습

반복 실패 패턴에서 새 하네스 규칙을 자동으로 제안한다.

```python
from hachillesworld.operate.meta_harness import MetaHarness

meta = MetaHarness(
    auto_apply_threshold=10,
    human_review_required=True   # 프로덕션에서 필수
)
```

## 8. SDK 통합 가이드

### 8.1 @instrument 데코레이터

```python
from hachillesworld import HAchillesWorldClient, instrument

client = HAchillesWorldClient()

@instrument(client, agent_name="supply-chain-agent")
class SupplyChainAgent:
    """plan / execute / observe / reflect 메서드가 자동 계측된다."""
    
    def plan(self, state: dict, goal: str) -> str:
        return self._select_action(state, goal)
    
    def execute(self, action: str) -> dict:
        return self._run_action(action)
    
    def observe(self, result: dict) -> dict:
        return self._update_state(result)
    
    def reflect(self, observation: dict) -> None:
        self._update_world_model(observation)
```

> `@instrument`는 `plan`, `execute`, `observe`, `reflect` 이름의 메서드만 계측한다.

## 15. 로그 형식 가이드

### 표준 이벤트 타입

| 이벤트 타입 | 의미 | 주요 payload 키 |
|-------------|------|----------------|
| `plan` | 다음 행동 계획 | `planning_depth`, `uncertainty`, `confidence` |
| `execute` | 환경 실행 | `action`, `cost_usd` |
| `observe` | 결과 관측 | `prediction_error`, `error_within_uncertainty`, `goal_achieved` |
| `reflect` | 반성·WM 업데이트 | `recalibrated`, `correction_applied`, `goal_changed` |
| `hitl_request` | 인간 개입 요청 | `reason`, `urgency` |
| `harness_violation` | 하네스 규칙 위반 시도 | `attempted_action`, `rule_id` |
| `checkpoint_recovery` | 체크포인트 복구 | `recovery_success`, `checkpoint_id` |
| `error` | 오류 발생 | `error_type`, `error_message` |

### 최소 필수 필드

```
필수:
  event_type  (str)  — "plan" / "execute" / "observe" / "reflect"

강력 권장:
  payload.planning_depth   (plan)     — Level 판정에 사용
  payload.prediction_error (observe)  — WMQ 측정에 사용
  payload.uncertainty      (plan)     — ECE·ALM 측정에 사용
  payload.recalibrated     (reflect)  — SDR 측정에 사용
  payload.goal_achieved    (observe)  — GAR 측정에 사용
```

### 기존 로그 변환

```python
def convert_logs(raw_logs: list[dict]) -> list[dict]:
    return [
        {
            "event_type": log.get("type", "unknown"),
            "timestamp":  log.get("ts", 0.0),
            "payload": {
                "planning_depth":   log.get("depth", 1),
                "prediction_error": log.get("pred_err", 0.0),
                "uncertainty":      log.get("unc", 0.0),
                "recalibrated":     log.get("recal", False),
                "goal_achieved":    log.get("goal_ok", False),
            }
        }
        for log in raw_logs
    ]
```

## 17. 자주 묻는 질문 (FAQ)

**Q: API 키 없이 사용할 수 있나요?**

A: 네. 로컬 진단(Scan), 로드맵 생성(Optimize), Drift 모니터링(Operate), HASInterpreter 모두 API 키 없이 동작합니다. API 키는 클라우드 이벤트 수집과 팀 공유 기능에만 필요합니다.

---

**Q: HAS 95% 신뢰구간이 너무 넓어요. 어떻게 좁힐 수 있나요?**

A: 신뢰구간 폭은 로그 데이터 양과 직결됩니다. 최소 50개 에피소드가 있어야 의미 있는 CI가 산출됩니다. `@instrument` 데코레이터로 로그를 충분히 쌓은 후 재진단하세요.

---

**Q: L2 판정을 받았는데 실제로는 L1 같아요.**

A: `planning_depth`가 로그에 없으면 기본값(1)이 사용돼 L1으로 낮게 판정됩니다. plan 이벤트의 payload에 `planning_depth` 키를 추가하세요.

---

**Q: CA 측정이 매번 다른 결과를 줘요. (v2.1 이전)**

A: v2.1부터 `HACHILLESWORLD_JUDGE_SEED`로 재현성을 제어합니다. 또한 LocalJudge 사용 시 외부 LLM API 의존성이 없어 결정론적 결과를 얻을 수 있습니다.

---

**Q: 에이전트 로그가 Anthropic 서버로 전송되나요?**

A: `HACHILLESWORLD_CA_JUDGE=local`(기본값) 설정 시 로그는 외부로 전송되지 않습니다. AnthropicJudge를 사용할 때만 CA 측정을 위한 로그 샘플이 Anthropic API로 전송됩니다. Privacy Filter로 사전에 민감정보를 제거하는 것을 권장합니다.

---

**Q: 여러 에이전트를 동시에 진단할 수 있나요?**

A: 네. `MultiAgentOptimizer`로 복수 에이전트를 분석하고, 조직 수준 포트폴리오 리포트와 리소스 배분 제안을 받을 수 있습니다. (§11 참조)

---

**Q: 3개월 전 HAS 74와 오늘 HAS 74가 같은 기준인가요?**

A: v2.1부터 가중치 버전이 리포트에 명시됩니다. `report.composite_score_at_version("버전")` 메서드로 동일 버전 기준으로 재산출해 비교할 수 있습니다.

---

**Q: Python 3.11 미만에서도 동작하나요?**

A: 지원하지 않습니다. Python 3.11 이상을 사용하세요.
