## 1. TRL(기술준비수준) 개요

IITP(정보통신기획평가원) 기준 TRL(Technology Readiness Level)은 1~9 단계로 구성되며,
본 과제는 **소프트웨어 기술** 분야 TRL 기준을 적용한다.

## 3. 현재 TRL 6 달성 근거 (SDK v2.0)

### 3.1 TRL 6 기준 충족 증거

**기준**: 운용 환경과 유사한 환경에서 시스템 시제품이 실증됨

| 항목 | 달성 내용 | 산출물 |
|------|----------|--------|
| 핵심 기능 완전 구현 | 15개 지표 전면 자동화 | Sprint 1-A~E |
| 운용 환경 모사 API | FastAPI REST API 서버 가동 | Sprint 3-A |
| 실사용 환경 파이프라인 | HAW-STUDY-001 실증 연구 (n=50) | Sprint 3-C, 4-B |
| 규제 준수 자동화 | EU AI Act·ISO 42001 자동 매핑 | Sprint 4-C |
| 멀티에이전트 지원 | 그룹 HAS·의존성 그래프 | Sprint 4-A |

### 3.2 SDK v1.1 → v2.0 TRL 진화 경로

```
SDK v1.1 (TRL 4)          SDK v2.0 (TRL 6)
─────────────────────     ──────────────────────────
12개 지표 자동화        →  15개 지표 완전 자동화
단독 에이전트 평가      →  멀티에이전트 그룹 HAS
CLI 단독 운용          →  REST API + 대시보드
수동 보고서            →  EU AI Act·ISO 자동 보고서
실험 환경 검증         →  실사용 유사 환경 실증 (n=50)
```

### 4.2 기술 수준 자가 평가 체계

HAchillesWorld SDK 자체를 활용한 SDK의 자가 품질 평가:

```python
# SDK 자가 평가 예시
from hachillesworld import HAchillesClient

client = HAchillesClient()
# SDK 테스트 에이전트 실행 로그로 SDK 품질 자가 진단
self_report = client.scan(agent_name="HAW-SDK-v2.0", ...)
print(f"SDK 자가 HAS: {self_report.composite_score:.1f}")
```
