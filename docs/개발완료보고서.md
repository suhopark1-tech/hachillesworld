## 1. 프로젝트 개요

### 1.1 목적

《Agentic World Modeling 2027》의 이론적 프레임워크를 실용적 소프트웨어 제품으로 구현한다. 기업과 개인이 AI 에이전트 시스템의 World Model 품질을 진단하고, 최적화하며, 운영할 수 있는 SDK + CLI + 분석 엔진을 제공한다.

### 1.2 핵심 테제

> "진단 없는 최적화는 도박이다."  
> HAchillesWorld는 Levels × Laws 프레임워크를 기반으로 에이전트를 진단하고, 다음 Level로의 구체적인 경로를 제시한다.

### 1.3 개발 기간

| 설계 | 2026-05-31 | 기획서 작성, 아키텍처 설계 |
| 구현 | 2026-06-01 | 전체 소스코드 개발 |

## 3. 아키텍처 설계 결정사항

### 3.1 핵심 설계 원칙

**원칙 1: 로컬 우선 (Local-first)**

API 서버 연결 없이 로컬에서 전체 진단·최적화 기능이 동작한다. 네트워크 의존성을 최소화해 엔터프라이즈 환경의 보안 정책과 호환된다.

```
Client.scan() → ScanEngine (로컬) → DiagnosticReport
                ↓ (API 서버 연결 시)
              /api/v1/scan → 클라우드 분석
```

**원칙 2: 계층형 데이터 모델**

모든 진단 결과는 불변(immutable) 데이터클래스로 표현된다. 직렬화·역직렬화가 용이하고, 상태 변이로 인한 버그를 방지한다.

```python
DiagnosticReport
  └── CategoryScore × 3 (World Model / 에이전시 / 운영)
       └── MetricScore × 5 (각 카테고리당)
```

**원칙 3: 확장 가능한 지표 시스템**

`MetricsCalculator`에 새 메서드를 추가하는 것만으로 새 지표를 등록할 수 있다. `ScanEngine`은 메서드 목록을 자동으로 수집하므로 중앙 등록 없이 지표가 활성화된다.

**원칙 4: 콜백 기반 경보**

`DriftMonitor.on_alert`는 함수를 주입받는다. Slack 알림, 데이터베이스 기록, 자동 재보정 트리거 등 다양한 경보 대상을 코드 수정 없이 교체할 수 있다.

### 3.2 Levels × Laws 진단 알고리즘

**Level 판정 로직**:

```
입력: wm_metrics (World Model 품질 지표), agency_metrics (에이전시 지표)

planning_depth   = wm_metrics["Planning Depth"].value
self_correction  = agency_metrics["Self-Correction Rate"].value
uncertainty_ok   = agency_metrics["Uncertainty Awareness"].value

if self_correction >= 0.30 AND planning_depth >= 20:
    → L3 Evolver
elif planning_depth >= 5 OR uncertainty_ok >= 1.0:
    → L2 Simulator
else:
    → L1 Predictor

level_progress = (지표 값 / 해당 Level 상한) × 9 → 소수점 표기 (예: L2.3)
```

**Laws 도메인 판정 로직**:

`agent_config["laws_domain"]` 키워드 매핑 방식. 키워드가 없으면 `Digital Laws`가 기본값.

```
"physical" / "robot"    → Physical Laws
"digital"  / "api"      → Digital Laws
"social"   / "multi"    → Social Laws
"science"  / "research" → Scientific Laws
```

### 3.3 하네스 자동 생성 설계

임계값을 초과한 지표 → 사전 정의된 매핑 테이블 → `HarnessRule` 객체 → Python 클래스 코드 생성의 파이프라인으로 동작한다.

```
MetricScore(status="critical")
  → HarnessGenerator._metric_to_rule(metric)
  → HarnessRule(condition, action, severity)
  → HarnessSpec.to_python()
  → 실행 가능한 Python 코드
```

**설계 의도**: 자동 생성된 규칙은 초안이다. 엔지니어가 검토하고 프로덕션에 적용하는 워크플로우를 강제함으로써 자동화의 위험을 제어한다.

## 4. 모듈별 개발 상세

### 4.1 Module 1: HAchillesWorld Scan

**파일**: `src/hachillesworld/scan/`

#### 4.1.1 MetricsCalculator (metrics.py)

15개 진단 지표를 3개 카테고리로 구현했다.

**Category A: World Model 품질 (5개)**

| 지표 | 구현 방법 | 정상 기준 |
|------|-----------|-----------|
| Prediction Error Rate | `observe` 이벤트의 `prediction_error` 평균 | < 0.15 |
| Calibration ECE | 10-구간 균등 캘리브레이션 오차 계산 | < 0.10 |
| Simulation Drift Rate | `reflect` 이벤트 중 `recalibrated=True` 비율 | < 5% |
| Uncertainty Coverage | `error_within_uncertainty=True` 비율 | > 80% |
| Planning Depth | `plan` 이벤트의 `planning_depth` 평균 | ≥ 5 steps |

**Category B: 에이전시 수준 (5개)**

| 지표 | 구현 방법 | 정상 기준 |
|------|-----------|-----------|
| Self-Correction Rate | `correction_applied=True` 비율 | ≥ 10% |
| Uncertainty Awareness | `uncertainty` 키 존재 여부 | 존재 |
| Goal Consistency | `goal_changed=True` 비율의 역수 | > 90% |
| Env Adaptation Speed | `recalibrated` 후 복구까지 스텝 수 | ≤ 10 steps |
| Harness Coverage | `config["harness_rules"]` 길이 | ≥ 20 |

**Category C: 운영 건전성 (5개)**

| 지표 | 구현 방법 | 정상 기준 |
|------|-----------|-----------|
| Recalibration Frequency | Drift Rate와 동일 | < 5% |
| Cost Efficiency | `cost_usd` 합산 / 예산 | ≤ 0.80 |
| HITL Trigger Rate | `hitl_request` 이벤트 비율 | < 5% |
| Harness Violation Rate | `harness_violation` 이벤트 수 | 0건/일 |
| Checkpoint Recovery Rate | `recovery_success=True` 비율 | > 98% |

#### 4.1.2 ScanEngine (engine.py)

- `run(logs, agent_name)` → `DiagnosticReport`
- 카테고리 점수: 지표별 status를 가중치(ok=1.0 / warning=0.5 / critical=0.0)로 변환해 0~100 점수 산출
- 종합 점수: World Model 40% + 에이전시 35% + 운영 25% 가중 평균

### 4.2 Module 2: HAchillesWorld Optimize

**파일**: `src/hachillesworld/optimize/`

#### 4.2.1 RoadmapGenerator (roadmap.py)

- `generate(report, target_level)` → `OptimizationRoadmap`
- L1→L2, L2→L3 두 가지 전환 경로에 대해 Phase 1~3 템플릿 사전 정의
- 각 Phase: 기간(주), 태스크 목록, 우선 태스크(★), 예상 점수 향상
- 비용 절감 추정: 예산 초과분 × 12개월

#### 4.2.2 HarnessGenerator (harness_generator.py)

- `generate(report)` → `HarnessSpec`
- 6개 지표에 대한 매핑 규칙 사전 정의
- `HarnessSpec.to_python()`: 실행 가능한 Python 클래스 코드 자동 생성
- 기본 금지 행동 4개 자동 포함
- 예산 상한 3개 계층(일/월/태스크) 자동 설정

### 4.3 Module 3: HAchillesWorld Operate

**파일**: `src/hachillesworld/operate/`

#### 4.3.1 DriftMonitor (monitor.py)

- 슬라이딩 윈도우(기본 20개) 기반 드리프트 이력 관리
- `record(predicted, actual)` → drift 스칼라 값 반환
- `on_alert` 콜백: 경보 임계율 초과 시 자동 호출
- `is_stable()`: 최근 윈도우의 드리프트율이 안전 범위 내인지 판단
- 두 딕셔너리 공통 숫자 키의 절댓값 차이 평균으로 drift 계산

#### 4.3.2 ReplayDebugger (replay.py)

- `load(episode_id, events)` → `ReplaySession`
- `AgentEvent` 객체 또는 딕셔너리 둘 다 입력 가능
- 이상 감지 규칙:
  - `observe` 이벤트: `prediction_error > 0.15` → 이상
  - `plan` 이벤트: `uncertainty > 0.35` → 이상
  - `error` 이벤트: 항상 이상
- 가장 이른 이상 프레임을 근본 원인으로 자동 판정
- `print_session()`: 터미널 컬러 출력 (이상 프레임 🔴 강조)

#### 4.3.3 MetaHarness (meta_harness.py)

- `record_failure(event)`: 실패 이벤트 누적, 패턴 키 기반 그룹화
- `auto_apply_threshold`: 이 횟수 이상 반복되면 규칙 제안 대기열에 추가
- `approve_rule / reject_rule`: 인간 검토 후 적용/거부
- `summary()`: 패턴 분포, 대기 규칙 수, 적용 규칙 수 요약

### 4.4 Core 모듈

**파일**: `src/hachillesworld/core/`

#### models.py
- `Level`, `LawsDomain`: Enum 타입 정의
- `MetricScore`: 단일 지표 (이름, 값, 임계값, 상태, 설명)
- `CategoryScore`: 카테고리 집계 (점수, 지표 목록)
- `DiagnosticReport`: 완전한 진단 리포트 (level_label, composite_score 등 프로퍼티 포함)
- `OptimizationRoadmap`, `RoadmapPhase`: 로드맵 구조
- `AgentEvent`: 이벤트 수집 단위

#### client.py — HAchillesWorldClient
- `scan(logs, config, agent_name)`: 로컬 ScanEngine 실행
- `emit(event)` / `flush()`: 이벤트 버퍼링 및 전송
- `track(agent_name, event_type, payload)`: 단순 이벤트 헬퍼
- Context Manager (`with` 구문) 지원

#### instrument.py
- `@instrument(client, agent_name)`: 클래스 데코레이터
- `plan / execute / observe / reflect` 4개 메서드 자동 래핑
- 각 메서드 호출 시 duration_ms, error 정보 자동 기록

#### config.py
- `pydantic-settings` 기반 환경변수 로드
- `HACHILLESWORLD_` 접두사로 모든 설정 주입 가능
- 기본값 포함 (API 키 없이도 로컬 실행 가능)
