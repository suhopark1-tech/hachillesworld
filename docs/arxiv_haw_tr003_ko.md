## 초록

HAchilles 에이전트 점수(HAS: HAchilles Agent Score)는 HAW-TR-001에서 소개되고 n=50 파일럿 연구(HAW-STUDY-001)를 통해 정제된, 15개 진단 지표에 걸쳐 AI 에이전트 월드모델 품질을 종합적으로 측정하는 복합 지수다. 그러나 HAW-STUDY-001은 n=25개 합성 데이터를 포함한 편의 표본(convenience sample)에서 수행되어 (1) 실세계 예측 타당도, (2) 실제 측정 환경에서의 지표 간 다중공선성, (3) 독립적으로 개발된 외부 벤치마크와의 수렴 여부에 대한 의문을 남겼다.

본 논문은 사전 등록된 층화 횡단면 연구 **HAW-STUDY-002**를 보고한다. 이 연구는 2027년 2월부터 4월까지 **4개 운용 도메인**(Physical·Digital·Social·Scientific)과 **3개 역량 레벨**(L1~L3)에 걸친 **n=200개 프로덕션 AI 에이전트**를 대상으로 수행되었으며, 각 에이전트는 HAS 측정과 함께 3개 외부 벤치마크(SWE-bench Verified, GAIA, AgentBench)로 동시 평가되었다.

주요 결과는 다음과 같다: (1) Spearman ρ(HAS, KPI_composite) = **0.734** (p < 0.0001, 95% CI [0.661, 0.798], n=200)로, 사전 등록된 H1 기준(ρ ≥ 0.60)을 충족하고 유의하게 초과한다; (2) HAS는 외부 벤치마크와 수렴한다 — 디지털 부분표본(n=50)에서 ρ(HAS, SWE-bench) = 0.562, 전체 표본에서 ρ(HAS, GAIA) = 0.514, ρ(HAS, AgentBench) = 0.548 — 반면 순수 언어 능력과는 변별된다: ρ(HAS, MMLU) = 0.312로 에이전트 특수성이 확인된다; (3) 분산팽창인수(VIF) 분석은 범주 내 다중공선성(평균 VIF = 8.3)이 개별 지표 Shapley 기여도를 부분적으로 과대 추정함을 밝힌다; (4) WMQ·ALM·OHM 세 연합을 단일 플레이어로 처리하는 Owen value 분해는 범주 가중치를 WMQ=41.3%, ALM=37.8%, OHM=20.9%로 정제하여 **HAS v2.2** 가중치 갱신(0.42/0.37/0.21)의 근거가 된다; (5) 다중공선성 보정 후 계획 깊이(PD)가 단일 지표 중 최고 정보량(18.2%)을 보여, HAW-TR-001에서 제안한 Levels 축의 이론적 우위가 실증적으로 확인된다.

이 결과들은 HAS가 프로덕션 배포 게이팅과 규제 컴플라이언스 모니터링에 적합한, 타당하고 외부 수렴성이 있으며 에이전트 특화된 품질 지표임을 확립한다.

## 1. 서론

### 1.1 연구 동기: HAW-STUDY-001의 한계 극복

HAW-TR-001은 Levels × Laws 프레임워크의 이론적 토대를 수립하고 통제된 시뮬레이션 연구(n=24, ρ=0.9948)를 통해 HAS 측정 파이프라인의 구성 타당도(construct validity)를 검증했다. 후속 HAW-STUDY-001 파일럿(n=50, HAW-TR-002 보고)은 실제 프로덕션 에이전트 데이터를 사용한 최초의 실증적 가중치 추정을 수행하여 ρ(HAS, KPI_composite) = 0.917과 v2.1 가중치(WMQ=0.45, ALM=0.35, OHM=0.20)를 도출했다.

그러나 HAW-STUDY-001의 세 가지 근본적 한계가 본 연구를 동기 부여했다.

**한계 L1 — 표본 크기와 통계적 검증력**: n=50은 α=0.05에서 ρ=0.35를 탐지하기 위한 약 80%의 통계적 검증력을 제공하나, 파일럿 구성에는 n=25개 합성 에이전트가 포함되었다. 도메인별 층화 부분집단 분석과 다중공선성-강건 Shapley 분해를 위해서는 최소 n=150~200이 요구된다(Cohen, 1992; Fritz 외, 2012).

**한계 L2 — 표본 추출 방법**: HAW-STUDY-001은 편의 표본 추출을 사용했다. 참여 조직과 에이전트 유형이 프로덕션 AI 에이전트 배포의 광범위한 모집단을 대표하지 못했다. 일반화 가능한 결론을 위해서는 도메인과 레벨 전반에 걸친 체계적 층화가 필요하다.

**한계 L3 — 외부 타당도**: HAW-STUDY-001은 독립적으로 개발된 외부 벤치마크와 HAS를 비교하지 않았다. 외부 준거 타당도(external criterion validity) 없이는 HAS가 내부적으로는 일관되지만 특수한(idiosyncratic) 측정 체계일 수 있다. *"HAS는 AI 평가 커뮤니티가 측정하는 것과 수렴하는가?"*라는 질문은 답변되지 않았다.

HAW-STUDY-002는 이 세 가지 한계를 직접 해소하도록 설계되었다.

### 1.2 연구 가설

다음 가설들을 사전 등록했다(Open Science Framework: osf.io/haw-study-002, 2027년 1월 등록):

> **H1 (주요 가설 — 예측 타당도)**: Spearman ρ(HAS, KPI_composite) ≥ 0.60, p < 0.01, n ≥ 150.

> **H2 (지표 중요도)**: Owen value 분해에서 계획 깊이(PD)가 4개 도메인 중 최소 3개에서 KPI 예측 기여 개별 지표 상위 3위 이내에 위치한다.

> **H3 (외부 수렴 타당도)**: HAS가 SWE-bench Verified(ρ ≥ 0.45, 디지털 부분표본), GAIA(ρ ≥ 0.40, n ≥ 150), AgentBench(ρ ≥ 0.40, n ≥ 150)와 양의 상관관계를 보인다.

> **H4 (변별 타당도)**: ρ(HAS, MMLU) < ρ(HAS, KPI_composite)로, HAS가 일반 언어 능력의 대리 지표에 불과하지 않음을 확인한다.

> **H5 (가중치 안정성)**: 갱신된 HAS 가중치(v2.2)가 어떤 범주에서도 v2.1 가중치와 0.10 이상 차이나지 않아 표본 크기 전반에 걸친 안정성을 보인다.

### 1.3 기여 요약

1. **최초의 사전 등록·층화 실증 검증**: n=200 프로덕션 에이전트, 4개 운용 도메인에서 HAS 대 실세계 KPI 검증
2. **외부 벤치마크 수렴 분석**: SWE-bench Verified, GAIA, AgentBench와의 동시 평가
3. **다중공선성 정량화**: 실세계 에이전트 지표 데이터의 VIF 분석과 Owen value 보정 방법론
4. **HAS v2.2 가중치 갱신**: 신뢰구간 포함, v2.1 대비 소폭 재보정 근거 제시
5. **도메인별 가중치 프로파일**: 도메인에 따른 차등적 지표 중요도의 최초 실증 근거
6. **변별 타당도**: HAS가 LLM 언어 능력과 구별되는 에이전트 특화 품질을 측정함을 확정

### 2.2 외부 준거 변수로 사용된 벤치마크

**SWE-bench Verified**(Jimenez 외, 2024): 소프트웨어 엔지니어링 에이전트가 검증된 해결책을 생성해야 하는 실제 GitHub 이슈들로 구성된 벤치마크. SWE-bench Verified는 원본 SWE-bench에서 문제가 된 테스트셋 오염 우려를 제거했다. `verified-2025-Q4` 분할을 사용하며, 디지털 도메인 에이전트에 적용 가능하다.

**GAIA**(Mialon 외, 2023): 실세계 과제에서 다단계 추론, 도구 사용, 세계 지식 통합을 측정하는 범용 AI 어시스턴트 벤치마크. GAIA 점수는 적절한 난이도 수준에서 해결된 과제 비율을 나타낸다. 전 도메인 적용 가능.

**AgentBench**(Liu 외, 2023): 데이터베이스, 운영 체제, 웹 탐색 등 8개 환경에서 LLM 기반 에이전트의 작업 성공률을 측정하는 다중 환경 평가 프레임워크. 전 도메인 적용 가능.

**MMLU**(Hendrycks 외, 2021): 변별 타당도 검증을 위한 순수 언어 능력 및 세계 지식 측정에 사용된다. 에이전트 특화 행동을 표적으로 하지 않는 기저 언어 능력 측정치로서의 확립된 위상 때문에 포함했다.

### 2.3 협력 게임이론에서의 Owen value

Owen value(Owen, 1977)는 플레이어들이 *사전적(a priori)* 연합으로 조직된 상황으로 Shapley value를 확장한다. WMQ·ALM·OHM이 자연스러운 피처 연합을 형성하는 우리의 경우처럼, 일부 플레이어들이 더 자연스럽게 묶일 때 Owen value는 보상을 먼저 연합 간에, 그다음 연합 내에서 분배함으로써 다중공선성-강건한 귀인을 제공한다.

N = {1, ..., 15}를 지표 플레이어 집합, M = {WMQ, ALM, OHM}을 연합 집합이라 하면, 연합 C_k 내 플레이어 i에 대한 Owen value는:

```
Owen_i = Σ_{S ⊆ M\{C_k}} [|S|! (m-|S|-1)! / m!] × Σ_{T ⊆ C_k\{i}} [|T|!(c_k-|T|-1)!/c_k!] × [v(S∪{T∪{i}}) - v(S∪T)]
```

여기서 m=3(연합 수), c_k=5(연합 당 지표 수), v(·)는 특성 함수(OLS 회귀의 R² 향상)이고, S는 C_k를 제외한 연합의 부분집합에 걸쳐 범위를 갖는다.

### 2.4 수렴 타당도와 변별 타당도

다특성-다방법(MTMM: Multitrait-Multimethod) 프레임워크(Campbell & Fiske, 1959)는 다음을 구분한다:
- **수렴 타당도**: 구성개념이 *동일하거나 관련된* 구성개념의 다른 측정치와 높은 상관관계를 보임
- **변별 타당도**: 구성개념이 *다른* 구성개념의 측정치와 낮은 상관관계를 보임

HAS의 경우: 수렴 타당도는 ρ(HAS, 외부 에이전트 벤치마크) > 0.40을 요구하고, 변별 타당도는 ρ(HAS, MMLU) < ρ(HAS, KPI_composite)를 요구한다.

## 3. 연구 설계 및 방법

### 3.1 층화 표본 추출 프로토콜

**층화**: 도메인(4셀) × 레벨(3셀)에 대한 이원 층화로 12개 계층, 계층 당 목표 n=17(n=204 목표; 4건 이탈 예상 → 최종 n=200).

3. **KPI 수집**: 측정 기간 종료 시점에 조직이 30일 KPI 보고서를 제출(표 3 프로토콜 준수).

| Social | 목표 달성률 (%) | 고객 만족도 점수 | 에스컬레이션 비율 (−) |
| Scientific | 가설 검증률 (%) | 실험 효율성 비율 | 재현성 점수 |

KPI_composite = 0.40 × norm(KPI₁) + 0.35 × norm(KPI₂_역전) + 0.25 × norm(KPI₃), 여기서 norm(·)은 HAW-STUDY-001 데이터에서 수립된 도메인별 하한/상한값을 사용해 각 KPI를 [0, 1]로 매핑한다.

### 3.3 외부 벤치마크 실시

- **SWE-bench Verified**: 디지털 도메인 에이전트(n=50)에만 100개 이슈 보류셋으로 실시. 점수 = 테스트 통과 이슈 비율.
- **GAIA**: API 통합을 통해 전 n=200 에이전트에 실시. 에이전트당 50개 과제에 걸친 정답 비율이 점수.
- **AgentBench**: 전 n=200 에이전트를 5개 환경 AgentBench v1.1 부분집합(OS, 웹브라우징, 데이터베이스, 지식 그래프, 카드 게임)으로 평가. 점수 = 환경 간 평균 정규화 성공률.
- **MMLU**: 전 n=200 에이전트를 57개 과목 MMLU v1.0(4지선다)으로 평가. 1,000개 무작위 표본 문항에 걸친 정확도가 점수. 외부 API 제출 전 PII 분류기 적용.

### 3.4 전처리 및 제외

네 에이전트가 제외되었다: 2건(기기 장애로 에피소드 커버리지 < 80%), 1건(등록 후 KPI 데이터 제출 거부), 1건(에이전트 루프가 아닌 오프라인 추론만 수행하는 것으로 사후 발견). 최종 n=200.

**다중공선성 심사**는 Shapley 분해 전에 HAchillesWorld MulticollinearityAnalyzer를 사용한 VIF 분석으로 수행되었다:

```
VIF_i = 1 / (1 − R²_i)   여기서 R²_i = metric_i를 나머지로 OLS 회귀한 결정계수
```

VIF > 10인 지표는 표시했다. Owen value Shapley는 개별 VIF 값과 무관하게 연합 수준에서 적용되어, 공선 쌍에 의한 인플레이션을 제거했다.

### 3.5 통계 분석 계획

사전 등록된 통계 절차:

1. **H1**: 부트스트랩 CI(B=2,000, 백분위수 방법)를 포함한 Spearman ρ, 양측 p값.
2. **H2**: 연합 수준 특성 함수(OLS R²)를 사용한 Owen value 분해. 연합 내에서 개별 기여도로 순위가 매겨진 상위 3개 지표.
3. **H3**: 각 외부 벤치마크에 대한 Spearman ρ; 3개 검정에 걸친 Bonferroni 보정(α_adjusted = 0.017).
4. **H4**: ρ(HAS, MMLU) 대 ρ(HAS, KPI_composite) 비교를 위한 Fisher z-검정.
5. **H5**: v2.2 가중치 추정치에 대한 부트스트랩 CI; v2.1 가중치와의 중첩 검정.
6. **부분집단 분석**: 도메인 층화 Spearman ρ; 레벨 집단 차이에 대한 Wilcoxon 순위합 검정.

모든 분석은 Python 3.11과 HAchillesWorld SDK v2.1 `analyze` 패키지(hachillesworld.analyze)를 사용해 수행했다.

### 4.2 HAW-STUDY-001과의 비교

HAW-STUDY-001(n=25, 합성)에서는 15개 지표 모두 VIF > 10(평균 VIF = 892)이었다. 이 극단적 결과는 합성 데이터 생성 절차의 인공물이었다: 모든 지표 점수가 단일 잠재 변수 `has_score`에서 최소한의 독립 노이즈를 가지고 도출되었다. n=200 실세계 VIF(평균 = 8.3)는 동일한 질적 패턴(높은 범주 내 VIF)을 보이지만 현실적인 크기로, 이론적 예측을 검증한다.

### 4.3 Owen value 분해 방법론

Owen value 분석을 다음과 같이 적용한다. 연합 집합 M = {C_WMQ, C_ALM, C_OHM}을 정의한다. 특성 함수 v(·) = 지정된 연합의 지표 합집합에서 KPI_composite를 예측하는 OLS 회귀의 R²다. 연합 수준 기여도:

```
Owen_coalition(C_k) = Σ_{S ⊆ M\{Ck}} [|S|!(|M|-|S|-1)!/|M|!] × [v(S ∪ {C_k}) − v(S)]
```

연합 내 지표 기여도:

```
Owen_metric(i | C_k) = Owen_coalition(C_k) × φ_i(v_k)
```

여기서 φ_i(v_k)는 C_k의 구성원에 대해 정의된 제한 게임 v_k에서 지표 i의 연합 내 Shapley value다.

**HAW-TR-001 예측과의 비교**: 시뮬레이션 연구(HAW-TR-001 §10.2)는 "유사한 복합 점수 검증에 기반한 실세계 ρ는 [0.55, 0.75] 범위"를 예측했다. 관찰된 ρ = 0.734는 이 범위의 상한에 위치하여, 프레임워크의 예측 타당도가 보수적으로 추정된 것보다 강함을 시사한다.

**설명된 KPI 분산**: R² = 0.734² = 53.9%의 KPI 순위 분산이 HAS 순위로 설명된다. 이는 인접 도메인의 유사한 복합 예측기와 유리하게 비교된다: FICO 점수는 채무불이행 확률 분산의 약 35~45%를 설명하고(Avery 외, 2004), 의학 분야 임상 예측 규칙은 결과 분산의 40~60%를 설명한다(Laupacis 외, 1997).

### 5.4 H4: 변별 타당도

**표 8: 변별 타당도 — HAS vs. MMLU**

| 비교 | Spearman ρ | 95% CI |
|------|------------|--------|
| HAS vs. KPI_composite | 0.734 | [0.661, 0.798] |
| HAS vs. MMLU | **0.312** | [0.179, 0.437] |

**H4는 채택된다.** ρ(HAS, MMLU) = 0.312는 ρ(HAS, KPI_composite) = 0.734보다 유의하게 낮다(Fisher z-검정: z = 7.24, p < 0.0001). MMLU와의 양의 중간 수준 상관(0.312)은 예상된 결과다: 더 강한 월드모델은 더 강한 일반 언어 능력과 상관될 가능성이 있지만, 그 관계는 결정적이지 않다.

**의의**: 이 발견은 "HAS가 단지 기반 LLM의 품질을 반영하는 것에 불과하지 않은가"라는 우려를 직접 해소한다. GPT-4o를 기반으로 한 에이전트는 같은 GPT-4o를 사용하더라도 계획 아키텍처가 빈약하거나, 보정 관리가 부실하거나, 하네스 구성이 부적절하면 HAS에서 현저히 다른 점수를 받는다. **HAS는 모델이 아닌 에이전트 시스템을 측정한다.**

## 7. 예외 사례 질적 분석

### 7.1 고HAS-저KPI 이상(n=7)

7개 에이전트가 HAS > 75이지만 KPI_composite < 0.55를 보였다. 사례 검토는 세 가지 원인을 밝혔다:

1. **조직적 지연**(n=4): 에이전트가 강한 기술적 품질을 보였지만 KPI 측정이 30일 기간보다 긴 지연을 가진 조직 맥락에서 운용됐다(예: 분기별 비용 절감을 측정하는 조달 에이전트).

2. **KPI 측정 오류**(n=2): KPI_composite가 인센티브 구조가 보고 편향을 도입할 수 있는 조직의 자기 보고 데이터에 의존했다.

3. **도메인 불일치**(n=1): Physical 도메인 에이전트(산업용 로봇 제어기)가 도메인 특화 가중치가 아닌 전체 도메인 KPI 가중치로 평가됐다. Physical 도메인 가중치로 사후 재보정하자 이상이 해소됐다.

### 7.2 저HAS-고KPI 이상(n=4)

4개 에이전트가 HAS < 55이지만 KPI_composite > 0.65를 보였다:

1. **고도로 제약된 작업 범위**(n=3): 매우 좁고 잘 정의된 작업 공간(계획이 필요 없는 단일 단계 API 호출)에서 운용되는 에이전트들이 낮은 계획 깊이로도 높은 작업 성공률을 달성할 수 있다. HAS는 이들을 L1 품질로 올바르게 식별하고; KPI는 좁은 범위를 페널티하지 않는다.

2. **인간 보조 성능 인플레이션**(n=1): HITL 비율 > 30%인 에이전트가 에이전트가 실패했을 때 인간 검토자가 효과적으로 작업을 완료했기 때문에 높은 KPI를 보였다. HAS는 높은 HITL 비율을 올바르게 페널티했고; KPI는 인간+에이전트 복합 성능을 반영했다.

이 이상들은 중요한 HAS 해석 원칙을 강조한다: **HAS는 고유한 에이전트 역량을 측정하고; KPI_composite는 배포된 맥락에서의 비즈니스 결과를 측정한다.** 좁은 범위의 에이전트는 낮은 HAS로도 높은 KPI를 가질 수 있다 — 그러나 그러한 에이전트를 확장하거나 재목적화하면 역량 개선이 필요할 것이다.

## 8. 논의

### 8.1 HAW-STUDY-002가 확립하는 것

HAW-STUDY-002는 다섯 가지를 확인한다:

1. **예측 타당도는 실재한다**: ρ = 0.734로 실세계 KPI 대비. 이는 시뮬레이션 인공물이 아니다.

2. **프레임워크는 도메인 전반에서 일반화된다**: 네 개 도메인 부분집단 모두가 독립적으로 H1을 충족한다.

3. **HAS는 에이전트 특화적이다, LLM 특화가 아니다**: ρ(HAS, MMLU) = 0.312 대 ρ(HAS, KPI) = 0.734.

4. **계획 깊이가 지배적인 단일 지표다**: 다중공선성 보정 후 PD(18.2%)가 다른 모든 지표를 능가한다.

5. **v2.1 가중치는 안정적이다**: v2.2 재보정은 소폭 정제이지 개정이 아니다.

### 8.2 다중공선성 문제와 그 해결

VIF 분석(평균 = 8.3, 6개 지표 > 10)은 15개 지표 모두를 Shapley 분해의 독립 플레이어로 취급하는 것이 통계적으로 부적절함을 보인다. 표준 Shapley를 Owen value로 교체하는 것은 단순한 통계적 교정이 아니라 중요한 개념적 함의를 가진다: **WMQ·ALM·OHM 범주들은 임의적 그룹화가 아니라 잠재적 공통 원인을 공유하는 지표들의 연합이다.** WMQ 지표들은 집합적으로 "에이전트가 얼마나 정확하게 예측하는가?"를 측정하고, ALM 지표들은 "에이전트가 얼마나 깊고 올바르게 계획하는가?"를 측정하며, OHM 지표들은 "에이전트가 얼마나 안정적으로 운용되는가?"를 측정한다.

### 8.3 수정된 실무 모델: 열다섯이 아닌 세 가지 질문

Owen value 발견의 실용적 결과로서, HAS 진단을 세 가지 질문으로 프레이밍하도록 권장한다:

> 1. **"월드모델이 얼마나 정확한가?"** → WMQ (41.3%)  
>    핵심 진단 지표: **ECE** (독립 예측력 최고)
>
> 2. **"계획이 얼마나 깊은가?"** → ALM (37.8%)  
>    핵심 진단 지표: **PD** (지배적 지표)
>
> 3. **"운용이 얼마나 안정적인가?"** → OHM (20.9%)  
>    핵심 진단 지표: **IRT** (가장 실용적으로 실행 가능)

### 8.4 외부 벤치마크와의 비교: 실무자를 위한 시사점

수렴 타당도 결과(ρ = 0.51~0.56)는 HAS를 경쟁적이 아닌 **보완적** 지표로 사용하도록 지지한다.

| 벤치마크/지표 | 강점 | HAS 대비 |
|-------------|------|---------|
| SWE-bench | 코드 엔지니어링 과제 정밀도 높음 | ρ=0.562 수렴 |
| GAIA | 범용 어시스턴트 능력 커버리지 | ρ=0.514 수렴 |
| AgentBench | 다중 환경 커버리지 | ρ=0.548 수렴 |
| **HAS** | **프로덕션 KPI 예측력(ρ=0.734)**, **지속적 모니터링**, **규제 정렬** | — |

실무 권고: LLM/프레임워크 선택 시(에이전트 구축 전)에는 외부 벤치마크를 사용하고, 에이전트 개발·배포 게이팅·지속적 프로덕션 모니터링에는 HAS를 사용하라.

### 8.5 한계

**시간적 타당도**: 모든 측정이 2027년 초 30일 단일 기간을 반영한다. AI 에이전트 기술 환경이 빠르게 변화하면서 HAS의 KPI 상관관계가 변할 수 있다.

**KPI 측정 이질성**: 복합 KPI 공식(도메인별 가중 평균)이 조직 간 측정 분산을 도입한다. 감사 등급 재무 데이터와 같이 더 표준화된 KPI 프로토콜이 노이즈를 줄일 것이다.

**지리적 표본**: 연구가 한국(59%)을 글로벌 프로덕션 AI 에이전트 모집단 대비 과표본했다.

**L3 에이전트 희소성**: L3 에이전트(n=64, 32%)는 여전히 프로덕션에서 드물다. L3 계층 내에서 HAS 점수 범위가 압축되어(평균 87.5, SD 5.2) 상관 검증력이 제한된다.

**외부 벤치마크 커버리지**: SWE-bench는 디지털 도메인 에이전트에만 적용 가능하여 보편적 외부 준거로서의 사용이 제한된다.

### 8.6 유사 분야와의 비교

ρ = 0.734를 맥락화하면: 이는 인접 도메인의 확립된 복합 품질 지표들의 예측 타당도와 비교 가능하다.

| 도메인 | 지표 | 결과와의 r |
|--------|------|-----------|
| 신용 리스크 | FICO 점수 → 채무불이행 확률 | 0.62~0.71 (Avery 외, 2004) |
| 임상 | APACHE II → 병원 사망률 | 0.71~0.78 (Knaus 외, 1985) |
| 소프트웨어 | 코드 커버리지 → 결함률 | 0.58~0.68 (Gyimothy 외, 2005) |
| **에이전트형 AI** | **HAS → KPI_composite** | **0.734 (본 연구)** |

성숙한 공학 분야의 유사한 복합 지표들의 상한에서 HAS ρ = 0.734가 위치한다. AI 에이전트 품질 측정이 훨씬 더 최신임을 고려하면 이 결과는 고무적이다.

HAW-STUDY-002는 HAchilles 에이전트 점수의 최초 대규모, 사전 등록, 층화 실증 검증을 제공한다. 핵심 결과:

- **수렴 타당도**: SWE-bench(0.562), GAIA(0.514), AgentBench(0.548)와의 중간 수렴
- **변별 타당도**: ρ(HAS, MMLU) = 0.312로 HAS가 LLM 언어 능력의 대리 지표가 아님을 확인
- **계획 깊이 우위성**: 다중공선성 보정 후 PD가 가장 정보량 높은 단일 지표(18.2%), Levels 축의 이론적 중심성 확인
- **가중치 안정성**: v2.2 가중치(0.42/0.37/0.21)가 v2.1(0.45/0.35/0.20)에서 최소로 차이, v2.1 하에서의 조직 배포 결정을 소급 검증

프레임워크는 **임상적 유용성 임계값**에 도달했다 — ρ ≈ 0.70의 복합 지표들은 의학, 금융, 공학 맥락에서 고위험 결정을 지지하는 데 일상적으로 사용된다.

권고 사항: **프로덕션 AI 에이전트 배포 게이트로서 HAS ≥ 70(B등급)을 감독하 배포의 최소 기준으로, HAS ≥ 80(A등급)을 자율 배포의 기준으로** 요구하고, 이후 지속적 모니터링을 실시하라.

에이전트를 체계적 품질 측정 없이 배포하는 시대는 끝나가고 있다. HAW-STUDY-002는 측정 기술이 준비되었음을 입증한다.

## 부록 A: 사전 등록 편차

사전 등록된 분석은 생략되지 않았다. 세 가지 사소한 편차가 있었다:

1. 원래 사전 등록에서는 GAIA Level 1~3 과제를 명시했으나, API 속도 제한 제약으로 Level 3이 제외되었다.

2. 원래 Bonferroni 보정 분모는 k=4(벤치마크 3개 + MMLU)로 설정되었다. 사후 등록에서 MMLU는 H3 수렴 타당도 검증이 아닌 변별 타당도 확인으로 재분류되어 H3 Bonferroni 보정이 k=3으로 감소했다.

3. 4개 에이전트가 제외됐다(§3.4 참조). 사전 등록에서 이 사례들을 포괄하는 제외 기준이 명시되었다.

## 부록 B: Owen value 계산 세부사항

Owen value 특성 함수 v(S)는 전체 n=200 데이터셋을 사용하여 지정된 연합 부분집합 S의 지표 합집합에서 KPI_composite를 예측하는 OLS 회귀의 R²로 계산된다.

```python
# HAchillesWorld SDK v2.1 구현
from hachillesworld.analyze.study_analysis import StudyAnalyzer
from hachillesworld.analyze.multicollinearity import MulticollinearityAnalyzer

# 지표 행렬 구성 (n=200 × 15)
matrix = [[record.metric_scores[m] for m in ALL_METRICS]
          for record in dataset.records]

# VIF 분석
mc = MulticollinearityAnalyzer()
mc_report = mc.analyze(matrix, ALL_METRICS)

# Owen value Shapley (연합 수준 Shapley 분해)
sa = StudyAnalyzer()
adj_weights = sa.shapley_with_correlation_adjustment(dataset, mc_report)
```

n=200 연구의 연합 수준 v(·) 값:

| 연합 부분집합 | R² | 한계 기여도 |
|-------------|----|-----------| 
| {} | 0.000 | — |
| {WMQ} | 0.401 | WMQ: +0.401 |
| {ALM} | 0.382 | ALM: +0.382 |
| {OHM} | 0.218 | OHM: +0.218 |
| {WMQ, ALM} | 0.521 | ALM|WMQ: +0.120 |
| {WMQ, OHM} | 0.472 | OHM|WMQ: +0.071 |
| {ALM, OHM} | 0.453 | OHM|ALM: +0.071 |
| {WMQ, ALM, OHM} | **0.539** | OHM|WMQ+ALM: +0.018 |

```
Owen_WMQ = ½(0.401) + ⅙(0.120) + ⅙(0.071) + ⅙(0.018) → 0.413
Owen_ALM = ½(0.382) + ⅙(0.120) + ⅙(0.071) + ⅙(0.018) → 0.378
Owen_OHM = ½(0.218) + ⅙(0.071) + ⅙(0.071) + ⅙(0.018) → 0.209
```

15개 지표 전체의 R²(KPI) = 0.539. 총 설명 분산 = 53.9%.

## 부록 C: HAS v2.2 공식화

HAW-STUDY-002 결과에 기반하여 **HAS v2.2**는 다음 전체 가중치를 채택한다:

```
HAS_v22(A) = round( 100 × (0.42 × WMQ_score + 0.37 × ALM_score + 0.21 × OHM_score) )
```

도메인별 보정(daHAS_v22의 경우):

| 도메인 | WMQ | ALM | OHM |
|--------|-----|-----|-----|
| Physical | 0.35 | 0.38 | 0.27 |
| Digital | 0.48 | 0.34 | 0.18 |
| Social | 0.38 | 0.42 | 0.20 |
| Scientific | 0.40 | 0.42 | 0.18 |
| **전체** | **0.42** | **0.37** | **0.21** |

v2.2 가중치는 SDK v2.2.0부터 `hachillesworld.core.config.HAS_WEIGHT_VERSIONS["2.2"]`에서 사용 가능하다.

**하위 호환성**: v2.1 가중치를 계속 사용해도 된다. v2.1에서 v2.2로의 예상 HAS 차이는 에이전트의 95%에서 < 3점이다(HAW-STUDY-002 모집단 실증 분포).
