## 1. 훈련 시나리오: 가상 P1 사고

```
사고명: HAW-DRILL-2026-P1
발생일: 훈련 당일 09:00 KST (UTC 00:00)
유형:   Level 1 다수 고객 데이터 유출 (가상)

가상 상황:
  · 공격자가 RDS 인증 취약점(가상)을 이용해 DiagnosticReport 테이블에
    무단 접근, EU 고객 12명(독일 7명, 아일랜드 5명)의
    이메일 주소(AES-256 암호화)와 에이전트 진단 수치 데이터에 접근

가상 데이터:
  · 영향 고객 수: 12명 (독일 7, 아일랜드 5)
  · 유출 항목: 이메일 주소(암호화됨), HAS 점수, 15개 지표값
  · 복호화 키 유출 여부: 없음 (암호화 유지)
  · 공격자 접근 시간: 2026-08-XX 03:15~03:42 UTC (27분)
  · 탐지 방법: AuditEvent 이상 패턴 → CloudWatch 알람 → CPO 이메일
```

## 2. 72시간 대응 체크리스트 (훈련 실행용)

각 단계를 실제로 수행하고 완료 시각을 기록한다.

### T+0 — 사고 탐지 (09:00 KST)

```
□ 최초 인지 시각 기록: 2026-08-XX 09:00 KST (00:00 UTC)
□ CPO(박성훈)에 즉시 보고
□ 내부 인시던트 채널 개설 (Slack #incident-DRILL-2026-P1)
완료 시각: ______________________
```

### T+1h — 초기 범위 파악 (10:00 KST)

```
□ EU 고객 포함 확인: YES (독일 7 + 아일랜드 5 = 12명)
□ 침해 데이터 유형 목록:
    - 이메일 주소 (AES-256 암호화, 복호화 키 미유출)
    - 에이전트 진단 수치 데이터 (HAS 점수, 15개 지표)
□ Art.33 신고 필요 여부 1차 판단: YES (EU 고객 + 개인정보 접근)
□ AuditEvent 조회 명령 실행:
    SELECT * FROM audit_events WHERE timestamp BETWEEN '03:00' AND '04:00' 
    AND action LIKE '%unauthorized%';
완료 시각: ______________________
```

### T+4h — 신고 준비 (13:00 KST)

```
□ Art.34 통지 필요 여부 판단:
    · 이메일 암호화 유지 → Art.34(3)(a) 예외 적용 → 통지 불필요
    · 수치 데이터 → 민감도 낮음 → 통지 불필요
    · 결론: Art.34 통지 불필요 (암호화 예외 해당)
□ HAW-IRP-EU-001 §5 신고 양식 초안 작성:
    Section A: HAchilles Labs / 박성훈
    Section B: Confidentiality breach / 12명 (EU 7+5)
    Section C: Low risk (암호화 유지, 복호화 키 미유출)
    Section D: 접근 권한 즉시 차단, 취약점 패치, 포렌식 조사 착수
□ 관할 DPA 확정:
    독일 7명 → BfDI (bfdi.bund.de/meldung)
    아일랜드 5명 → DPC (dataprotection.ie/en/report-a-breach)
    → 2개 DPA에 동시 신고 필요
완료 시각: ______________________
```

### T+24h — 신고 초안 완성 (다음날 09:00 KST)

```
□ BfDI 신고 양식 영어본 완성 (HAW-IRP-EU-001 §5 템플릿 기반)
□ DPC 신고 양식 영어본 완성
□ 두 DPA 포털 계정 접속 확인:
    · BfDI: meldungen.bfdi.bund.de 로그인 테스트
    · DPC:  dataprotection.ie 계정 확인
□ 침해 고객 국가별 이메일 목록 준비 (실제 훈련에서는 가상 이메일 사용)
완료 시각: ______________________
```
